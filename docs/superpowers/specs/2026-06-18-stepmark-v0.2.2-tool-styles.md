## 1. 范围概述

本 spec 对应 V0.2.2 版本,聚焦**让用户自定义每个工具的样式**。用户调一次,后续所有新建标注都用新样式,且重启保留。

### 1.1 包含

| 编号 | 内容 |
|------|------|
| B1 | 每工具独立样式:颜色 / 线宽 / 形状 / 线型 / 字体族 / 字号 / 箭头头大小 |
| B2 | 样式持久化(localStorage,带校验和默认回退) |
| B3 | 工具栏单击工具 → 下方二级选项面板(Snipaste 风格) |
| B4 | 样式只影响**新建**标注,不回改已建标注 |
| B5 | 字体缺失回退、损坏设置回退默认 |

### 1.2 不包含(留给后续)

- 已建标注的样式修改(双击改字已有,改颜色/线宽留 V0.3)
- 多套样式预设(profile 切换)—— V0.3
- 主窗口的"设置页"(V0.2.2 只用工具栏二级面板,符合 PRD §Implementation Boundary)
- AI 分析、问题清单导出

## 2. 数据模型

### 2.1 ToolStyleSettings(新增)

`src/types/toolStyle.ts`:

```typescript
import type { ShapeKind } from "./annotation";  // V0.2.1 新增

export type LineStyle = "solid" | "dashed";

// 每工具独立的样式参数。只含该工具实际用得到的字段。
export interface SmartToolStyle {
  color: string;          // 矩形/箭头/标签背景色(三者同色,跟 V0.1 一致)
  strokeWidth: number;    // 线宽 1-8
  shape: ShapeKind;       // "rect" | "ellipse"(V0.2.1 已支持渲染)
  fontSize: number;       // 标签字号
  fontFamily: string;     // 标签字体族
}

export interface RectToolStyle {
  color: string;
  strokeWidth: number;
  shape: ShapeKind;
}

export interface ArrowToolStyle {
  color: string;
  strokeWidth: number;
  lineStyle: LineStyle;   // "solid" | "dashed"
  arrowHeadSize: number;  // 箭头头尺寸 4-20
}

export interface TextToolStyle {
  color: string;          // 文字颜色(不是背景色;文字工具无背景)
  fontSize: number;
  fontFamily: string;
}

export interface ToolStyleSettings {
  smart: SmartToolStyle;
  rect: RectToolStyle;
  arrow: ArrowToolStyle;
  text: TextToolStyle;
}

// 注意:mosaic 工具不含样式配置(马赛克是像素化,没有颜色/线宽概念)。
```

### 2.2 默认值(对照 PRD:保持 V0.1 视觉)

`src/types/toolStyle.ts`:

```typescript
export const DEFAULT_TOOL_STYLES: ToolStyleSettings = {
  smart: {
    color: "#ff4757",
    strokeWidth: 3,
    shape: "rect",
    fontSize: 13,
    fontFamily: "",  // 空字符串 = 跟随系统(V0.1 行为)
  },
  rect: {
    color: "#ff4757",
    strokeWidth: 3,
    shape: "rect",
  },
  arrow: {
    color: "#ff4757",
    strokeWidth: 3,
    lineStyle: "solid",
    arrowHeadSize: 10,
  },
  text: {
    color: "#ff4757",   // 文字本身红色(跟 V0.1 标签文字色一致)
    fontSize: 13,
    fontFamily: "",
  },
};
```

### 2.3 与现有 AnnotationStyle 的关系

V0.1 的 `AnnotationStyle`(每个标注内嵌的样式)**保留不动**。新建标注时,把工具样式映射成一组标注字段(部分进 `AnnotationStyle`,部分进 V0.2.2 新增的扩展字段),固化进 annotation。

`bgColor` 的处理需要说明:只有 smart 工具的标签有背景(背景色 = 主色);矩形/箭头/文字标注没有背景填充。所以 `bgColor` 字段对 smart 设为主色,对其他工具设为主色(V0.1 渲染层对非 smart 标注不会读 `bgColor`,设了也无害,但保持 V0.1 行为不破)。

### 2.4 Annotation 类型扩展

`src/types/annotation.ts` 在 V0.2.1 基础上再加:

```typescript
export interface Annotation {
  id: string;
  type: ToolType;
  rect?: Rect;
  shape?: ShapeKind;         // V0.2.1 已加(smart/rect 用)

  // V0.2.2 新增:
  lineStyle?: LineStyle;     // arrow 用
  arrowHeadSize?: number;    // arrow 用
  fontFamily?: string;       // smart/text 用(空 = 跟随系统)

  note?: string;
  arrow?: ArrowData;
  style: AnnotationStyle;    // 含 color/strokeWidth/fontSize(沿用 V0.1)
}
```

> 为什么不把所有样式塞进 `style` 对象?—— 保持 V0.1 的 `AnnotationStyle` 结构不变,避免破坏 V0.1 已有的渲染代码。新字段是 optional,旧标注(undefined)按默认行为渲染。

## 3. 持久化(B2)

### 3.1 存储位置

`localStorage`,key = `"stepmark.toolStyles.v1"`。

> 版本号后缀(`.v1`)是为了将来 schema 变更时能检测旧格式并迁移/丢弃。

### 3.2 存储内容

整个 `ToolStyleSettings` 对象的 JSON。

### 3.3 加载 + 校验(对照 PRD §Edge Cases:corrupt settings fallback)

`src/store/toolStyleStore.ts`:

```typescript
import { create } from "zustand";
import { DEFAULT_TOOL_STYLES, type ToolStyleSettings } from "../types/toolStyle";
import type { SmartToolStyle, RectToolStyle, ArrowToolStyle, TextToolStyle } from "../types/toolStyle";

const STORAGE_KEY = "stepmark.toolStyles.v1";

function validateString(v: unknown): v is string {
  return typeof v === "string" && v.length > 0;
}
function validateColor(v: unknown): boolean {
  return typeof v === "string" && /^#[0-9a-fA-F]{6}$/.test(v);
}
function validateInt(v: unknown, min: number, max: number): v is number {
  return typeof v === "number" && Number.isInteger(v) && v >= min && v <= max;
}

// 每个工具的字段级校验,失败回退默认。
function validateSmart(v: unknown): SmartToolStyle {
  const d = DEFAULT_TOOL_STYLES.smart;
  if (typeof v !== "object" || v === null) return { ...d };
  const o = v as Partial<SmartToolStyle>;
  return {
    color: validateColor(o.color) ? o.color : d.color,
    strokeWidth: validateInt(o.strokeWidth, 1, 8) ? o.strokeWidth : d.strokeWidth,
    shape: o.shape === "rect" || o.shape === "ellipse" ? o.shape : d.shape,
    fontSize: validateInt(o.fontSize, 8, 72) ? o.fontSize : d.fontSize,
    fontFamily: typeof o.fontFamily === "string" ? o.fontFamily : d.fontFamily,
  };
}
// validateRect / validateArrow / validateText 类似,略(写进 plan 时补全)

function loadSettings(): ToolStyleSettings {
  try {
    const raw = localStorage.getItem(STORAGE_KEY);
    if (!raw) return { ...DEFAULT_TOOL_STYLES };
    const parsed = JSON.parse(raw);
    if (typeof parsed !== "object" || parsed === null) return { ...DEFAULT_TOOL_STYLES };
    const p = parsed as Partial<ToolStyleSettings>;
    return {
      smart: validateSmart(p.smart),
      rect: validateRect(p.rect),
      arrow: validateArrow(p.arrow),
      text: validateText(p.text),
    };
  } catch {
    return { ...DEFAULT_TOOL_STYLES };
  }
}

interface ToolStyleState {
  settings: ToolStyleSettings;
  updateTool: <K extends keyof ToolStyleSettings>(
    tool: K,
    patch: Partial<ToolStyleSettings[K]>
  ) => void;
  resetTool: (tool: keyof ToolStyleSettings) => void;
}

export const useToolStyleStore = create<ToolStyleState>((set, get) => ({
  settings: loadSettings(),
  updateTool: (tool, patch) => {
    const next = { ...get().settings, [tool]: { ...get().settings[tool], ...patch } };
    try {
      localStorage.setItem(STORAGE_KEY, JSON.stringify(next));
    } catch (e) {
      console.error("Failed to persist tool styles", e);
    }
    set({ settings: next });
  },
  resetTool: (tool) => {
    const next = { ...get().settings, [tool]: DEFAULT_TOOL_STYLES[tool] };
    try {
      localStorage.setItem(STORAGE_KEY, JSON.stringify(next));
    } catch (e) {
      console.error("Failed to persist tool styles", e);
    }
    set({ settings: next });
  },
}));
```

### 3.4 字体可用性回退(对照 PRD §Edge Cases)

字体回退**不在存储层做**,在**渲染层**做。Konva `Text` 接受任意 `fontFamily`,浏览器自动回退到系统字体。所以即使 `fontFamily` 存了 "Comic Sans MS" 而系统没有,渲染也不会崩,只是显示成系统默认。

**唯一需要处理的**:如果 `fontFamily === ""`(空字符串)→ 渲染时不设 `fontFamily` 属性,让 Konva 用默认(=跟随系统)。这是 V0.1 的行为,保持兼容。

## 4. 工具栏二级面板(B3)

### 4.1 触发

单击工具栏上的工具按钮 → 切换到该工具(V0.1 行为) + **同时在该按钮下方展开二级面板**。

> 用户 brainstorming 中确认:不是右键/长按,是**单击就展开**(Snipaste 同款)。这跟 brainstorming 早先说的"右键/长按"有出入,以最后一次确认为准 —— 单击展开。

再次单击同一工具按钮 → 收起面板。

### 4.2 面板位置

面板是 HTML 元素(不是 Konva),绝对定位在工具栏正下方,水平居中对齐被点击的工具按钮。

```
┌─────────────────────────────────────────┐
│  [⭐智能] [▭矩形] [➜箭头] [T文字] ...    │  ← 工具栏
│      └──────────────┐                  │
│                    ↓                   │
│      ┌──────────────────┐              │
│      │ 颜色: ●●●●●●      │  ← 二级面板  │
│      │ 线宽: ─── 3px     │             │
│      │ 形状: ▭ ○         │             │
│      │ 字号: ─── 13      │             │
│      │ 字体: [系统 ▾]    │             │
│      └──────────────────┘              │
└─────────────────────────────────────────┘
```

### 4.3 各工具面板内容

| 工具 | 面板控件 |
|------|---------|
| 智能标注 | 颜色色板 / 线宽滑块(1-8) / 形状(rect/ellipse 二选一) / 字号滑块(8-72) / 字体下拉 |
| 矩形 | 颜色色板 / 线宽滑块 / 形状(rect/ellipse) |
| 箭头 | 颜色色板 / 线宽滑块 / 线型(solid/dashed) / 箭头头大小滑块(4-20) |
| 文字 | 颜色色板 / 字号滑块 / 字体下拉 |
| 马赛克 | (无面板) |
| 复制/保存/✕ | (无面板,这些是动作按钮不是工具) |

### 4.4 控件规范

**颜色色板**:

```
[红] [蓝] [绿] [黄] [紫] [黑] [白] [自定义...]
```

- 8 个预设色块 + 1 个自定义(点开调色板,跟 V0.2.2 用浏览器原生 `<input type="color">`)
- 预设色:
  ```typescript
  const PRESET_COLORS = ["#ff4757", "#1890ff", "#52c41a", "#faad14", "#722ed1", "#000000", "#ffffff", "#fa8c16"];
  ```
- 当前选中色:色块外圈高亮(2px 蓝边)

**线宽滑块**:`<input type="range" min="1" max="8" step="1">`,旁边显示数值 + 实时预览(一条当前色当前粗细的线段)

**形状切换**:两个图标按钮 `▭` / `○`,当前选中高亮

**线型切换**:两个图标按钮 `━━`(实线) / `╌╌`(虚线)

**字号滑块**:`<input type="range" min="8" max="72" step="1">` + 数值

**箭头头大小滑块**:`<input type="range" min="4" max="20" step="1">` + 数值

**字体下拉**:`<select>` 列出系统常见字体 + "跟随系统"选项:
```typescript
const FONT_OPTIONS = [
  { label: "跟随系统", value: "" },
  { label: "微软雅黑", value: "Microsoft YaHei" },
  { label: "宋体", value: "SimSun" },
  { label: "黑体", value: "SimHei" },
```

## 5. 样式应用到新建标注(B4)

### 5.1 创建时读取工具样式

每个 `use*Tool.ts` 在构造 `Annotation` 时,把 `useToolStyleStore` 的当前工具样式映射进去:

```typescript
// useRectTool.ts 示例
import { useToolStyleStore } from "../store/toolStyleStore";
import { styleForNewAnnotation } from "../style/styleMapping";

// onMouseUp 创建标注时:
const settings = useToolStyleStore.getState().settings;
const a: Annotation = {
  id: crypto.randomUUID(),
  type: "rect",
  rect: preview,
  shape: settings.rect.shape,           // 新增
  style: styleForNewAnnotation("rect", settings),
};
addAnnotation(a);
```

`styleForNewAnnotation` 不够用(它只返回 `AnnotationStyle`),需要扩展为返回完整的标注扩展字段。重构成 `annotationFieldsFromToolStyle`:

```typescript
// src/style/styleMapping.ts
import type { Annotation, AnnotationStyle } from "../types/annotation";
import type { ToolStyleSettings } from "../types/toolStyle";

export function annotationFieldsFromToolStyle(
  tool: "smart" | "rect" | "arrow" | "text",
  settings: ToolStyleSettings
): Pick<Annotation, "style" | "shape" | "lineStyle" | "arrowHeadSize" | "fontFamily"> {
  const t = settings[tool];

  // textColor 规则:smart 标签文字始终白(对比度);text 工具文字 = 主色;
  // rect/arrow 不显示文字,textColor 用不上,但为安全设为主色。
  const textColor = tool === "smart" ? "#ffffff" : t.color;

  // bgColor 规则:smart 标签背景 = 主色;其他工具无背景填充,渲染层不读此字段,
  // 设为主色仅为保持 V0.1 数据形状不变(避免 undefined)。
  const bgColor = t.color;

  const style: AnnotationStyle = {
    borderColor: t.color,
    borderWidth: t.strokeWidth,
    bgColor,
    textColor,
    fontSize: "fontSize" in t ? t.fontSize : 13,
  };

  return {
    style,
    shape: "shape" in t ? t.shape : undefined,
    lineStyle: "lineStyle" in t ? t.lineStyle : undefined,
    arrowHeadSize: "arrowHeadSize" in t ? t.arrowHeadSize : undefined,
    fontFamily: "fontFamily" in t ? t.fontFamily : undefined,
  };
}
```

> 删除本节前面那个简版 `styleForNewAnnotation` —— 它是过渡说明,实际只用 `annotationFieldsFromToolStyle` 这一个。

### 5.2 已建标注不回改

**明确**:用户在面板改颜色,只影响**之后**画的标注。已画在画布上的标注保持原样(因为它们的样式已经固化进 `annotation.style`)。

> 这符合 PRD §Edge Cases:"V0.2.0 only requires style settings to affect newly created annotations."
>
> 已建标注改样式是 V0.3 的事(需要"选中标注 → 应用当前样式"的显式动作)。

### 5.3 渲染层适配

shape 组件需要读取标注上的新字段:

- `BoundableShape`(V0.2.1)已根据 `annotation.shape` 渲染 rect/ellipse,**无需改**
- `ArrowShape` 新增:根据 `annotation.lineStyle` 切 `dash`(solid=`[]`,dashed=`[10,6]`);根据 `annotation.arrowHeadSize` 设 `pointerLength/pointerWidth`
- `TextLabelShape` 新增:根据 `annotation.fontFamily` 设 Konva Text 的 `fontFamily`(空则不设)

```typescript
// ArrowShape.tsx
<Arrow
  points={...}
  stroke={a.style.borderColor}
  strokeWidth={a.style.borderWidth}
  dash={a.lineStyle === "dashed" ? [10, 6] : undefined}
  fill={a.style.borderColor}
  pointerLength={a.arrowHeadSize ?? 10}
  pointerWidth={a.arrowHeadSize ?? 10}
/>
```

## 7. 交互细节

### 7.1 面板展开/收起不影响工具切换

- 单击工具 A → 切到 A + 展开 A 的面板
- 单击工具 B → 切到 B + 展开 B 的面板(自动关闭 A 的)
- 再次单击当前工具 B → 收起面板(工具不变,还是 B)
- 点画布开始画标注 → 面板保持展开(不收起,用户可能要边画边调)

### 7.2 面板遮挡处理

工具栏在屏幕底部时,面板默认向上展开(不向下)。逻辑跟工具栏位置翻转复用 V0.1 代码(`flipUp` 判断)。

### 7.3 颜色色板的"自定义"

点"自定义"色块 → 弹出浏览器原生 `<input type="color">`(隐藏 input,点色块触发 click)。选完写入 `updateTool(tool, { color })`。

### 7.4 线宽/字号滑块的"滚轮 + 键盘"(对照 PRD §Input/Output)

滑块支持:
- 鼠标拖动(原生 range)
- 滚轮:`onWheel` 阻止默认滚动,`deltaY < 0` 加 1,`> 0` 减 1,clamp 到范围
- 键盘:聚焦后 ← → 微调 1,Shift+← → 大步 5

- [ ] 改完样式,关闭 StepMark,重新打开,样式保留
- [ ] 手动改坏 localStorage(改 JSON 为非法值),重启后回退默认,不崩溃

## 9. 风险与缓解

| 风险 | 影响 | 缓解 |
|------|------|------|
| 已建标注样式固化,用户改了面板以为会全局生效 | 体验困惑 | 面板顶部加一行小字提示:"仅影响新标注"(具体文案写进 plan) |
| localStorage 在 Tauri WebView2 中被禁用或配额满 | 持久化失效 | `setItem` 包 try/catch,失败仅 console.warn,不阻断使用 |
| 测量文字宽度(Konva Text)与实际渲染不一致 | 标签框宽偏差 | 用 V0.2.1 已验证的 `measureWidth` 函数,复用 |
| 面板控件太多导致拥挤 | 视觉乱 | 每工具只显示该工具用得到的控件(§4.3),不统一塞 |
| 字体下拉硬编码可能遗漏用户常用字体 | 体验 | 提供"自定义"入口(可手动输入字体名)—— V0.3 再做,V0.2.2 用硬编码列表 |

3. PRD 未提"面板顶部提示仅影响新标注",本 spec 加了(避免用户困惑)。
