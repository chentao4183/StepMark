## 1. 范围概述

V0.3.0 聚焦一个核心差异点:**自动序号标注**。用户绘制智能标注、矩形/椭圆、箭头、文字时,可自动生成全局递增的独立编号块,用于截图反馈中的问题编号和说明引用。

### 1.1 包含

| 编号 | 内容 |
|------|------|
| C1 | 智能标注 / 矩形 / 箭头 / 文字支持自动编号 |
| C2 | 编号在同一张截图内全局递增,每张截图从 1 开始 |
| C3 | 编号设置持久化,但 `nextNumber` 不跨截图持久化 |
| C4 | 编号块是独立 badge,不是文本内容的一部分 |
| C5 | 每工具独立开关和位置配置 |
| C6 | 编号块全局一套样式:背景色 / 文字色 / 形状 / 字号 |
| C7 | 删除不重排,undo/redo 保持编号确定性 |

### 1.2 不包含

- 马赛克工具自动编号。
- 编号块单独选中、拖动、缩放。
- 删除后自动重排编号。
- 跨截图连续编号。
- 每工具独立编号样式。
- 编号块边框颜色、边框宽度。
- 主窗口设置页。

## 2. 核心规则

### 2.1 全局递增

同一个 editor session 只有一个 `nextNumber`:

```typescript
interface EditorState {
  nextNumber: number;
}
```

示例:

1. 画智能标注,生成 badge `1`。
2. 开启矩形编号,画矩形,生成 badge `2`。
3. 开启箭头编号,画箭头,生成 badge `3`。

工具类型不影响编号序列。

### 2.2 生命周期

- 每次新截图进入编辑器时,`nextNumber = 1`。
- `nextNumber` 存在 `editorStore`,属于当前编辑会话。
- 编号开关、位置、样式存在持久化 store,跨 app 重启保留。
- `nextNumber` 不写入 localStorage。

### 2.3 默认值

| 项 | 默认值 |
|----|--------|
| 智能标注自动编号 | 开启 |
| 矩形自动编号 | 关闭 |
| 箭头自动编号 | 关闭 |
| 文字自动编号 | 关闭 |
| 智能标注默认位置 | 标签左侧 |
| 矩形默认位置 | 矩形左上 |
| 椭圆默认位置 | 椭圆左侧 |
| 箭头默认位置 | 终点 |
| 文字默认位置 | 左侧 |
| badge 背景色 | `#1677ff` |
| badge 文字色 | `#ffffff` |
| badge 形状 | 方形 |
| badge 字号 | `13` |

### 2.4 删除 / undo / redo

- 删除编号 `2` 后,编号 `3` 仍然是 `3`。
- undo 新建带编号标注时,恢复创建前的 `nextNumber`。
- redo 新建带编号标注时,恢复原标注和原编号值。
- undo/redo 删除操作只恢复/删除对应标注,不重排其他 badge。
- "重置为 1"只影响之后新建的 badge,不改已有 badge。
- "重置为 1"不进入 annotation undo/redo 历史,因为它不修改画布,只影响未来创建。

## 3. 数据模型

### 3.1 Annotation 扩展

`src/types/annotation.ts`:

```typescript
export type BadgeShape = "square" | "rounded" | "circle";

export interface NumberBadgeStyle {
  bgColor: string;
  textColor: string;
  shape: BadgeShape;
  fontSize: number;
}

export interface NumberBadge {
  value: number;
  style: NumberBadgeStyle;
}

export interface Annotation {
  id: string;
  type: ToolType;
  rect?: Rect;
  shape?: ShapeKind;
  lineStyle?: LineStyle;
  arrowHeadSize?: number;
  fontFamily?: string;
  note?: string;
  arrow?: ArrowData;
  style: AnnotationStyle;

  // V0.3.0 新增。没有该字段表示该标注无编号。
  numberBadge?: NumberBadge;
}
```

`NumberBadge.style` 是创建时固化的快照。用户之后改全局 badge 样式,不回改已有标注。

### 3.2 位置类型

新增 `src/types/numbering.ts`:

```typescript
export type NumberedTool = "smart" | "rect" | "arrow" | "text";

export type RectBadgePosition =
  | "top-left"
  | "top-right"
  | "bottom-left"
  | "bottom-right"
  | "center";

export type EllipseBadgePosition = "left" | "right" | "top" | "bottom" | "center";

export type ArrowBadgePosition = "start" | "middle" | "end";

export type TextBadgePosition = "left" | "right";

export type SmartBadgeAnchor = "target" | "arrow" | "label";

export interface SmartBadgePlacement {
  anchor: SmartBadgeAnchor;
  targetRectPosition: RectBadgePosition;
  targetEllipsePosition: EllipseBadgePosition;
  arrowPosition: ArrowBadgePosition;
  labelPosition: TextBadgePosition;
}

export interface TargetBadgePlacement {
  rectPosition: RectBadgePosition;
  ellipsePosition: EllipseBadgePosition;
}

export interface NumberingSettings {
  enabledByTool: Record<NumberedTool, boolean>;
  positionByTool: {
    smart: SmartBadgePlacement;
    rect: TargetBadgePlacement;
    arrow: ArrowBadgePosition;
    text: TextBadgePosition;
  };
  badgeStyle: NumberBadgeStyle;
}
```

`NumberedTool` 是固定 4-key union,不要从 `ToolType` 派生。`ToolType` 还包含 `select` 和 `mosaic`,这两个不参与自动编号。

### 3.3 持久化默认值

```typescript
export const DEFAULT_NUMBERING_SETTINGS: NumberingSettings = {
  enabledByTool: {
    smart: true,
    rect: false,
    arrow: false,
    text: false,
  },
  positionByTool: {
    smart: {
      anchor: "label",
      targetRectPosition: "top-left",
      targetEllipsePosition: "left",
      arrowPosition: "end",
      labelPosition: "left",
    },
    rect: {
      rectPosition: "top-left",
      ellipsePosition: "left",
    },
    arrow: "end",
    text: "left",
  },
  badgeStyle: {
    bgColor: "#1677ff",
    textColor: "#ffffff",
    shape: "square",
    fontSize: 13,
  },
};
```

存储 key:

```typescript
const STORAGE_KEY = "stepmark.numbering.v1";
```

校验规则:

- 颜色: `^#[0-9a-fA-F]{6}$`
- 字号: 整数,范围 `8-72`
- shape: `square | rounded | circle`
- enabled: 必须是 boolean
- placement: 必须是对应 union 的成员
- 任一字段非法时只回退该字段默认值,不要因为局部损坏丢弃整份设置。

## 4. Editor Store 变更

### 4.1 Snapshot

`src/store/editorStore.ts` 的历史快照加入 `nextNumber`:

```typescript
interface EditorSnapshot {
  annotations: Annotation[];
  cropRegion: Rect;
  nextNumber: number;
}
```

`snapshot(s)` 必须复制 `nextNumber`。

### 4.2 State / actions

新增:

```typescript
interface EditorState {
  nextNumber: number;
  resetNextNumber: () => void;
}
```

`init(...)` 时设置:

```typescript
nextNumber: 1
```

推荐把编号消费放在 `addAnnotation` 内部,避免各工具 hook 分散维护:

```typescript
addAnnotation: (a: Annotation, options?: { consumeNumber?: boolean }) => void;
```

若实现不想改签名,也可以提供专用 action:

```typescript
addNumberedAnnotation: (a: Annotation) => void;
```

约束:

- 新建带 badge 的 annotation 时,history 里记录创建前的 `nextNumber`。
- annotation 入库后,`nextNumber += 1`。
- 不带 badge 的 annotation 不改变 `nextNumber`。
- `removeAnnotation` 不改变 `nextNumber`。
- `updateAnnotation` 不改变 `nextNumber`,除非未来版本显式支持手动改编号。
- `resetNextNumber` 不写入 undo/redo history。

`toolState.ts` 是绘制过程中的临时跨组件状态,不负责持久化设置;V0.3 应新增独立 `numberingStore.ts`,不要合并进 `toolState.ts`。

## 5. 编号应用流程

新增 `src/numbering/applyNumbering.ts`:

```typescript
export function applyNumberBadgeIfEnabled(
  tool: NumberedTool,
  annotation: Annotation,
  settings: NumberingSettings,
  nextNumber: number
): { annotation: Annotation; consumed: boolean } {
  if (!settings.enabledByTool[tool]) {
    return { annotation, consumed: false };
  }

  return {
    annotation: {
      ...annotation,
      numberBadge: {
        value: nextNumber,
        style: { ...settings.badgeStyle },
      },
    },
    consumed: true,
  };
}
```

工具 hook 创建 annotation 后调用该 helper,再把结果交给 store。

注意:

- 禁用编号的工具不消耗编号。
- 马赛克不调用该 helper。
- 已有标注编辑时不重新应用编号。

## 6. Badge 几何

新增 `src/geometry/numberBadge.ts`。所有函数保持纯函数,方便单测。

### 6.1 尺寸

```typescript
export interface BadgeBox {
  width: number;
  height: number;
}

export const BADGE_HORIZONTAL_PADDING = 6;
export const BADGE_VERTICAL_PADDING = 3;
export const BADGE_MIN_SIZE = 18;
export const BADGE_ANCHOR_GAP = 4;
```

```typescript
export function measureNumberBadge(
  value: number,
  style: NumberBadgeStyle,
  measureText: (text: string, fontSize: number) => number
): BadgeBox
```

规则:

- 文本 = `String(value)`。
- 方形: `size = max(textWidth + paddingX * 2, fontSize + paddingY * 2, BADGE_MIN_SIZE)`,宽高都为 `size`。
- 圆形:同方形,渲染为圆。
- 圆角矩形:宽高分别计算,都不小于 `BADGE_MIN_SIZE`。

### 6.2 边界内移

```typescript
export function keepRectInsideCrop(rect: Rect, crop: Rect): Rect
```

规则:

- 若 badge 比 crop 更宽,`x = crop.x`。
- 若 badge 比 crop 更高,`y = crop.y`。
- 否则 clamp 到 `[crop.x, crop.x + crop.width - rect.width]` 和 `[crop.y, crop.y + crop.height - rect.height]`。

### 6.3 矩形位置

```typescript
export function rectBadgeBox(rect: Rect, box: BadgeBox, position: RectBadgePosition, crop: Rect): Rect
```

位置:

- `top-left`:矩形左上外侧。
- `top-right`:矩形右上外侧。
- `bottom-left`:矩形左下外侧。
- `bottom-right`:矩形右下外侧。
- `center`:矩形内部居中。

四角优先外侧放置,若出 crop 则 `keepRectInsideCrop`。

### 6.4 椭圆位置

```typescript
export function ellipseBadgeBox(bounds: Rect, box: BadgeBox, position: EllipseBadgePosition, crop: Rect): Rect
```

位置:

- `left`:椭圆左侧,垂直居中。
- `right`:椭圆右侧,垂直居中。
- `top`:椭圆上方,水平居中。
- `bottom`:椭圆下方,水平居中。
- `center`:椭圆中心。

left/right/top/bottom 优先贴椭圆外侧 `BADGE_ANCHOR_GAP`,再内移。

### 6.5 箭头位置

```typescript
export function arrowBadgeBox(arrow: ArrowData, box: BadgeBox, position: ArrowBadgePosition, crop: Rect): Rect
```

规则:

- `start`:以 `(startX,startY)` 为中心。
- `middle`:以 start/end 中点为中心。
- `end`:以 `(endX,endY)` 为中心。
- 若 `startX/startY` 缺失,用 `endX/endY` 回退,避免旧数据崩溃。

### 6.6 文本位置

```typescript
export interface TextBox {
  x: number;
  y: number;
  width: number;
  height: number;
}

export function textBadgeBox(textBox: TextBox, box: BadgeBox, position: TextBadgePosition, crop: Rect): Rect
```

规则:

- `left`:badge 右边缘在文本框左边缘左侧 `BADGE_ANCHOR_GAP`。
- `right`:badge 左边缘在文本框右边缘右侧 `BADGE_ANCHOR_GAP`。
- 位置依据渲染后的 visual text box,不要把编号拼入文本内容。

### 6.7 智能标注位置

```typescript
export function smartBadgeBox(
  annotation: Annotation,
  placement: SmartBadgePlacement,
  labelBox: TextBox,
  badgeBox: BadgeBox,
  crop: Rect
): Rect
```

规则:

- `placement.anchor === "target"`:
  - `annotation.shape === "ellipse"` 时使用 `placement.targetEllipsePosition`。
  - 其他情况使用 `placement.targetRectPosition`。
- `placement.anchor === "arrow"`:调用 `arrowBadgeBox`。
- `placement.anchor === "label"`:调用 `textBadgeBox`,默认左侧。
- 如果缺少 `rect` 或 `arrow`,回退到 label box 左侧;再缺失则不渲染 badge。

## 7. 渲染

### 7.1 NumberBadgeShape

新增 `src/canvas/shapes/NumberBadgeShape.tsx`:

```typescript
interface NumberBadgeShapeProps {
  badge: NumberBadge;
  box: Rect;
}
```

渲染:

- `square`:Konva `Rect`,`cornerRadius=0`。
- `rounded`:Konva `Rect`,`cornerRadius=4`。
- `circle`:Konva `Circle`,半径 = `min(width,height)/2`。
- 文本居中,颜色 = `badge.style.textColor`,字号 = `badge.style.fontSize`。
- 不渲染边框。

### 7.2 Shape 集成

以下组件在基础标注之后渲染 badge:

- `src/canvas/shapes/SmartAnnotationGroup.tsx`
- `src/canvas/shapes/RectShape.tsx`
- `src/canvas/shapes/ArrowShape.tsx`
- `src/canvas/shapes/TextLabelShape.tsx`

约束:

- badge 在视觉上位于标注上层。
- badge 使用 source space 计算,再跟随 V0.2 坐标转换进入 viewport/stage。
- badge 不参与选中框和拉手计算。
- 导出必须包含 badge。因为 badge 是 Konva node,正常导出 annotation layer 时应自然包含。

## 8. UI

### 8.1 入口

复用 V0.2.2 的 `StylePanel`。不新增主设置页。

每个支持的工具面板加入:

- 自动编号开关。
- 该工具的位置选择。
- 全局 badge 样式控件。
- 重置为 1。

### 8.2 控件建议

为避免面板过宽,可把编号控件做成紧凑分组:

```text
编号 [开关] 位置 [分段控件...] 样式 [色块/形状/字号] [重置1]
```

若一次塞不下,允许拆成两行,但不要新开独立设置窗口。

### 8.3 各工具面板

智能标注:

- 自动编号开关。
- anchor:目标 / 箭头 / 标签。
- anchor=目标:
  - shape=rect:左上 / 右上 / 左下 / 右下 / 居中。
  - shape=ellipse:左 / 右 / 上 / 下 / 居中。
- anchor=箭头:起点 / 中点 / 终点。
- anchor=标签:左 / 右。

矩形:

- 自动编号开关。
- shape=rect:左上 / 右上 / 左下 / 右下 / 居中。
- shape=ellipse:左 / 右 / 上 / 下 / 居中。

箭头:

- 自动编号开关。
- 起点 / 中点 / 终点。

文字:

- 自动编号开关。
- 左 / 右。

全局 badge 样式:

- 背景色。
- 文字色。
- 形状:方形 / 圆角 / 圆形。
- 字号:8-72。

## 9. 验收标准

- [ ] 新截图进入编辑器时,`nextNumber` 从 1 开始。
- [ ] 智能标注默认自动生成 badge 1。
- [ ] 矩形、箭头、文字默认不自动编号。
- [ ] 开启矩形/箭头/文字编号后,它们与智能标注共用全局递增序列。
- [ ] 禁用编号的工具不消耗编号。
- [ ] 删除带编号标注后,其他编号不重排。
- [ ] undo 新建带编号标注后,nextNumber 回退。
- [ ] redo 后恢复原 badge 值。
- [ ] 重置为 1 不修改已有 badge。
- [ ] 智能标注支持 target/arrow/label 三类 anchor。
- [ ] 矩形目标支持四角 + 居中。
- [ ] 椭圆目标支持左/右/上/下/居中。
- [ ] 箭头支持起点/中点/终点。
- [ ] 文字支持左/右。
- [ ] badge 支持背景色、文字色、方形/圆角/圆形、字号。
- [ ] badge 靠近 crop 边缘时自动内移,导出可见。
- [ ] 复制到剪贴板和保存图片都包含 badge。
- [ ] 损坏 localStorage 后重启不崩溃,回退默认设置。

## 10. 风险与约束

| 风险 | 等级 | 处理 |
|------|------|------|
| 智能标注组合 anchor 多,位置逻辑分散 | 中 | 所有 badge 几何集中到 `geometry/numberBadge.ts` |
| undo/redo 后 nextNumber 错乱 | 中 | history snapshot 纳入 `nextNumber` |
| HTML badge 无法导出 | 高 | badge 必须用 Konva 渲染 |
| StylePanel 过宽 | 中 | 编号控件紧凑分组,必要时两行 |
| 多位数 badge 过小 | 低 | measure + padding,圆角矩形允许变宽 |
