## 2. 架构变更:全屏坐标系统(核心)

这是本 spec 最重要也最有风险的部分。所有其他改动都依赖它。

### 2.1 现状(V0.1)

```
F1 截图 → 选区窗口裁剪出 crop 图片 → 编辑器只拿到裁后的图
编辑器内所有坐标 = 裁后图的本地坐标 (原点在裁后图左上角)
```

**问题**:用户进编辑器后无法调整 crop(原图已被丢弃,没东西可扩)。

### 2.2 目标(V0.2.1)

```
F1 截图 → 选区窗口裁剪 → 选区窗口把 {全屏原图 base64, 初始 crop 矩形} 都传给编辑器
编辑器内所有坐标 = 全屏原图坐标 (原点在全屏原图左上角)
编辑器渲染:在全屏原图上画一个 crop 视口,只显示 crop 区域内的内容
crop 可调 → 拉手拖动改变 crop 矩形 → 视口内容实时更新
```

### 2.3 坐标空间定义

整个编辑器只使用**一个坐标空间:全屏原图坐标(source space)**。

| 概念 | 定义 |
|------|------|
| `sourceImage` | 全屏截图 PNG base64(整个主显示器) |
| `cropRegion` | 在 source space 中的矩形 `{x, y, width, height}`,表示当前可见区域 |
| `annotation.rect.x/y` | source space 坐标(不是裁后图本地坐标) |
| `annotation.arrow.endX/endY` | source space 坐标 |
| Konva Stage 坐标 | source space(背景图以 `(-cropRegion.x, -cropRegion.y)` 放置,等于把 crop 左上角对齐到 stage 原点) |

### 2.4 坐标转换辅助函数

新建 `src/geometry/coordinates.ts`(纯函数,可单测):

```typescript
import type { Rect } from "../types/annotation";

// source space ↔ Konva stage space 的转换。
// Konva stage 的 (0,0) = cropRegion 的左上角。
// 所以:stage = source - cropOrigin,source = stage + cropOrigin。

export function sourceToStage(p: { x: number; y: number }, crop: Rect): { x: number; y: number } {
  return { x: p.x - crop.x, y: p.y - crop.y };
}

export function stageToSource(p: { x: number; y: number }, crop: Rect): { x: number; y: number } {
  return { x: p.x + crop.x, y: p.y + crop.y };
}

// Konva 鼠标事件给出的 pointer position 是 stage space。
// 工具 hook 拿到鼠标位置后必须先转成 source space 再存入 annotation。
export function pointerToSource(
  pointerStage: { x: number; y: number },
  crop: Rect
): { x: number; y: number } {
  return stageToSource(pointerStage, crop);
}
```

### 2.5 渲染策略

编辑器画布分三层(Konva Layer):

1. **BackgroundLayer**:渲染 `sourceImage`,位置 `(-cropRegion.x, -cropRegion.y)`。整张全屏图都在 stage 上,但被 `clip` 函数裁掉 crop 外的部分(见 §2.6)。
2. **AnnotationLayer**:所有标注,坐标全部是 source space,跟 BackgroundLayer 同步偏移。
3. **OverlayLayer**:crop 拉手、预览矩形、选中虚线框等交互辅助。

由于 Background 和 Annotation 用同一个偏移,标注永远跟背景对齐 —— 拖 crop 时背景和标注一起移动,视觉上像"窗口在图上滑动"。

### 2.6 clip 实现

Konva 用 `clipX / clipY / clipWidth / clipHeight` 属性在 Group/Stage 上裁剪。BackgroundLayer 和 AnnotationLayer 都设置 clip 为 crop 在 stage space 的范围:

```typescript
<Layer
  clipX={0}
  clipY={0}
  clipWidth={cropRegion.width}
  clipHeight={cropRegion.height}
>
  {/* 内容坐标都是 stage space (= source - cropOrigin) */}
</Layer>
```

crop 外的标注(用户缩了 crop 把标注框挤出可见区)**仍然存在于 store,只是被 clip 隐藏**(符合 PRD §Edge Cases 要求)。导出时也按 crop 裁掉。

## 5. Bug #2 & #3:智能标注的标签与箭头

### 5.1 Bug #3:箭头起点沿边滑动 + 外移

#### 算法

鼠标在 source space 的位置 `m`,矩形 `r`:

1. 计算鼠标到矩形**最近边上的最近点** `p`(已有函数 `nearestRectEdgePoint`)。
2. 把 `p` 沿"矩形中心 → p"方向**外移**一个固定量 `OUTSET = 6px`,得到 `p'`。
3. 箭头起点 = `p'`。

新建 `src/geometry/arrowAnchor.ts`:

```typescript
import type { Rect } from "../types/annotation";
import { nearestRectEdgePoint } from "./rectEdge";

const OUTSET = 6;  // 外移像素

// 智能标注箭头起点:沿最近边滑动 + 略微外移。
export function smartArrowStart(rect: Rect, mouse: { x: number; y: number }): { x: number; y: number } {
  const p = nearestRectEdgePoint(rect, mouse);
  // 沿 rect 中心 → p 的方向外移
  const cx = rect.x + rect.width / 2;
  const cy = rect.y + rect.height / 2;
  const dx = p.x - cx;
  const dy = p.y - cy;
  const len = Math.hypot(dx, dy);
  if (len === 0) return p;
  return {
    x: p.x + (dx / len) * OUTSET,
    y: p.y + (dy / len) * OUTSET,
  };
}

// 椭圆版本的箭头起点(椭圆 smart 标注用)
export function smartEllipseArrowStart(
  rect: Rect,  // 椭圆的 bounding box
  mouse: { x: number; y: number }
): { x: number; y: number } {
  const cx = rect.x + rect.width / 2;
  const cy = rect.y + rect.height / 2;
  const rx = rect.width / 2;
  const ry = rect.height / 2;
  // 椭圆参数方程:中心 → mouse 方向与椭圆边界的交点
  const dx = mouse.x - cx;
  const dy = mouse.y - cy;
  if (dx === 0 && dy === 0) return { x: cx, y: cy };
  // t 使 (cx + t*dx, cy + t*dy) 落在椭圆上
  const t = 1 / Math.sqrt((dx * dx) / (rx * rx) + (dy * dy) / (ry * ry));
  const edgeX = cx + t * dx;
  const edgeY = cy + t * dy;
  // 外移
  const len = Math.hypot(dx, dy);
  return {
    x: edgeX + (dx / len) * OUTSET,
    y: edgeY + (dy / len) * OUTSET,
  };
}
```

#### 数据模型影响

智能标注的 `arrow.startCorner` 字段**废弃**(改回绝对坐标 `startX/startY`)。因为起点现在实时沿边滑动,存角标识没意义了 —— 角标识适合"固定角"场景,沿边滑动必须存实时计算的绝对坐标。

但**已创建的标注**(用户松手定终点那一刻)需要把当时的起点存下来:

```typescript
// useSmartAnnotationTool 在用户"点定终点"时
const start = smartArrowStart(rect, mouseAtPinMoment);  // 计算一次
annotation.arrow = {
  startX: start.x,
  startY: start.y,
  endX: pinX,
  endY: pinY,
  labelX: ...,
  labelY: ...,
};
```

之后移动矩形时,起点**跟随矩形移动**(整体平移,见 §7.2),不重新计算 —— 已创建的标注是静态的,只有"正在画的预览"才实时滑动。

### 5.2 Bug #2:标签延伸方向 + 右对齐输入

#### 标签位置规则

标签锚点 = `arrow.endX/endY`(或 `labelX/labelY` 如果有)。标签朝**远离矩形**方向延伸:

- 计算锚点相对于矩形的位置:锚点在矩形左侧 → 标签向左延伸;右侧 → 向右;上 → 向上;下 → 向下。
- 简化为左右二分(因为标签是横向文本):如果 `anchor.x < rect.x + rect.width/2`(锚点在矩形中线左侧)→ 标签向左延伸,否则向右。

新建 `src/geometry/labelBox.ts` 已有 V0.1 的版本,需扩展支持左右二分:

```typescript
export type LabelSide = "left" | "right";

// 判断标签在矩形哪一侧
export function labelSide(
  anchor: { x: number; y: number },
  rect: { x: number; width: number }
): LabelSide {
  return anchor.x < rect.x + rect.width / 2 ? "left" : "right";
}

// 根据侧别计算 label 框左上角坐标(锚点为参考点)
export function labelBoxPosition(
  anchor: { x: number; y: number },
  side: LabelSide,
  boxWidth: number,
  boxHeight: number
): { x: number; y: number } {
  // y 方向:标签在锚点上方(box top = anchor.y - boxHeight),留 2px 间距
  const y = anchor.y - boxHeight - 2;
  if (side === "right") {
    // 向右延伸:锚点在标签左侧
    return { x: anchor.x, y };
  } else {
    // 向左延伸:锚点在标签右侧
    return { x: anchor.x - boxWidth, y };
  }
}
```

#### 输入框右对齐(关键)

`src/components/TextInputOverlay.tsx` 增加 `align` prop:

```typescript
interface Props {
  x: number;
  y: number;
  align: "left" | "right";  // 新增
  initial: string;
  onSubmit: (text: string) => void;
  onCancel: () => void;
}
```

- `align="left"`:输入框原点在左侧,文字左→右输入(V0.1 行为)。
- `align="right"`:
  - `text-align: right`(不使用 `direction: rtl` —— rtl 在中文标点处会错位)
  - 输入框的 `left` 坐标随文字增长向左移动 —— 用一个隐藏的 measure span 测量文字宽度,动态调整 input 的 `left = anchor.x - measuredWidth`,且 input 的 `width` 也跟着变(让 `text-align:right` 生效)。
  - 视觉上:锚点(箭头终点)固定在输入框右边缘,文字从锚点向左生长。光标自然跟随(中文输入法下光标在已输入文字末尾,即框右侧)。

实现细节(写进 plan,spec 里只定行为):

```typescript
// TextInputOverlay 右对齐时的核心逻辑
const [text, setText] = useState(initial);
const measureRef = useRef<HTMLSpanElement>(null);
const [width, setWidth] = useState(60);

useEffect(() => {
  if (measureRef.current) {
    setWidth(Math.max(60, measureRef.current.offsetWidth + 24));  // +padding
  }
}, [text]);

const style: React.CSSProperties = align === "right"
  ? { left: x - width, top: y, width, textAlign: "right", direction: "rtl" }
  : { left: x, top: y };

return (
  <>
    {align === "right" && (
      <span ref={measureRef} style={{ position: "absolute", visibility: "hidden", whiteSpace: "pre", font: "inherit" }}>
        {text || " "}
      </span>
    )}
    <input style={style} value={text} onChange={(e) => setText(e.target.value)} ... />
  </>
);
```

## 6. 标注缩放(A6)

### 6.1 可缩放的标注

| 标注类型 | 缩放方式 |
|---------|---------|
| 矩形 / 智能标注 / 马赛克 | 拖 8 拉手改 `rect` |
| 椭圆(smart/rect 的 ellipse 形状) | 拖 8 拉手改 bounding box `rect` |
| 箭头 | **不**用 8 拉手;改为在两端各显示一个圆形端点手柄,拖动改 `startX/Y` 或 `endX/Y` |
| 文字 | **不**可缩放(用 V0.2.2 的字号设置改大小) |

### 6.2 8 拉手的缩放语义

bounding box = `{x, y, width, height}`。8 拉手对应:

| 拉手位置 | 拖动效果 |
|---------|---------|
| 左上角 | 改 x、y、width、height(右下角不动) |
| 顶边中点 | 改 y、height |
| 右上角 | 改 y、width、height |
| 左边中点 | 改 x、width |
| 右边中点 | 改 width |
| 左下角 | 改 x、width、height |
| 底边中点 | 改 height |
| 右下角 | 改 width、height |

**约束**:缩放后 `width >= 10 && height >= 10`(最小尺寸),否则忽略此次拖动。

### 6.3 椭圆缩放

椭圆用 bounding box 表示,缩放 bounding box 即可。渲染时根据 `shape === "ellipse"` 用 `Ellipse` 组件,中心 = bounding box 中心,半径 = width/2, height/2。

### 6.4 缩放对智能标注的影响

智能标注 = 矩形(或椭圆) + 箭头 + 标签。缩放 rect 时:
- **箭头起点不重新计算**(已创建的标注是静态的,见 §5.1)
- **箭头终点和标签位置不动**(用户调的是矩形,不是注释指向)

也就是说,缩放智能标注的矩形,可能让箭头起点离开矩形边(视觉上箭头悬空)。这是**可接受的折衷** —— 重新计算起点会改变用户原本定的指向,反而更怪。如果用户介意,可以删了重画。

> 备选方案(不采纳,记录给 V0.3):缩放智能标注时同步按比例移动箭头终点和标签。但这破坏"标签位置是用户独立定的"语义。本 spec 不做。

## 7. Crop 调整(A5)

### 7.1 触发与位置

进编辑器后,**默认就可调整 crop**(不需要切换工具)。crop 拉手始终显示在 OverlayLayer,位于当前 crop 边界的 4 边中点 + 4 角。

### 7.2 拉手语义(对照 Snipaste)

| 拉手 | 拖动效果 |
|------|---------|
| 左边中点 | 改 crop.x、crop.width(向左扩或向右缩) |
| 右边中点 | 改 crop.width |
| 顶边中点 | 改 crop.y、crop.height |
| 底边中点 | 改 crop.height |
| 四角 | 同时改两个维度 |

### 7.3 约束(对照 PRD §Edge Cases)

```typescript
function clampCrop(next: Rect, source: { width: number; height: number }): Rect {
  const MIN = 20;
  // 不能超出原图
  let x = Math.max(0, next.x);
  let y = Math.max(0, next.y);
  let right = Math.min(source.width, next.x + next.width);
  let bottom = Math.min(source.height, next.y + next.height);
  let width = right - x;
  let height = bottom - y;
  // 最小尺寸
  if (width < MIN) {
    // 试图向右扩到 MIN
    if (x + MIN <= source.width) width = MIN;
    else { x = source.width - MIN; width = MIN; }
  }
  if (height < MIN) {
    if (y + MIN <= source.height) height = MIN;
    else { y = source.height - MIN; height = MIN; }
  }
  return { x, y, width, height };
}
```

放到 `src/geometry/crop.ts`,带单测。

### 7.4 crop 与标注的坐标关系

**关键**:标注坐标永远是 source space,crop 变化**不影响**标注的存储坐标。crop 缩小把标注挤出可见区 → 标注被 clip 隐藏但仍存在;crop 放大 → 之前隐藏的标注重新可见。

### 7.5 crop 平移(对照 Snipaste "拖 crop 内部移动整个选区")

Snipaste 支持拖 crop 内部空白区域平移整个 crop。本 spec **纳入**:
- 在 crop 内部、非标注区域按下拖动 → 平移 crop(改 x/y,不改 width/height)
- 平移同样 clamp 到原图边界

### 7.6 crop 进 undo/redo

`EditorSnapshot` 包含 `cropRegion`(见 §3.1)。每次 crop 变化(拉手松手、平移松手)压一次栈。

### 7.7 crop 调整 vs 标注交互的冲突处理

crop 拉手在 OverlayLayer(最上层),标注在 AnnotationLayer。冲突点:

- 点 crop 拉手 → 调 crop
- 点 crop 内部空白 → 平移 crop
- 点标注 → 选中标注(标注层的事件优先级要高于 crop 内部平移)

实现:crop 平移热区是一个透明 Rect,覆盖 crop 区域。**z-order 在标注之下**(AnnotationLayer 在 OverlayLayer 之下,Konva 默认下层先收到事件)。

但 Konva 的事件是"命中优先于层级":鼠标点哪个 shape 就触发哪个 shape 的事件,不会自动冒泡到下层。所以平移热区放在 OverlayLayer、但用 `listening` 切换:

- 默认情况下,工具处于 "select" 模式时,平移热区 `listening=true`(拦截 crop 内部点击);
- 当鼠标 hover 到某个标注时,标注临时 `listening=true` 优先于热区接收点击;
- 其他工具(智能标注、矩形等)激活时,平移热区 `listening=false`(让工具 hook 在 AnnotationLayer 之上工作)。

简化方案(本 spec 采纳):平移热区永远 `listening=false`,改用 Stage 级 `mousedown` 事件判断 —— 如果 `mousedown` 落在 crop 内且未命中任何标注 → 进入平移。这样不依赖 Konva 层级,逻辑集中在一个 handler。

export default function BoundableShape({ a }: { a: Annotation }) {
  if (!a.rect) return null;
  if (a.shape === "ellipse") {
    return (
      <Ellipse
        x={a.rect.x + a.rect.width / 2}
        y={a.rect.y + a.rect.height / 2}
        radiusX={a.rect.width / 2}
        radiusY={a.rect.height / 2}

  }
  return (
    <Rect
      x={a.rect.x}
      y={a.rect.y}

      height={a.rect.height}
      stroke={a.style.borderColor}
      strokeWidth={a.style.borderWidth}
    />
  );

### 8.3 椭圆几何

新建 `src/geometry/ellipse.ts`:

```typescript
import type { Rect } from "../types/annotation";

// 椭圆边界上,从中心指向 external 点方向的交点。
export function ellipseBoundaryPoint(
  bbox: Rect,
  external: { x: number; y: number }
): { x: number; y: number } {
  const cx = bbox.x + bbox.width / 2;
  const cy = bbox.y + bbox.height / 2;
  const rx = bbox.width / 2;
  const ry = bbox.height / 2;
  const dx = external.x - cx;
  const dy = external.y - cy;
  if (dx === 0 && dy === 0) return { x: cx, y: cy };
  const t = 1 / Math.sqrt((dx * dx) / (rx * rx) + (dy * dy) / (ry * ry));
  return { x: cx + t * dx, y: cy + t * dy };
}
```
