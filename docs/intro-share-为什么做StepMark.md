# 截图反馈问题,为什么还在切 3 次工具?我做了一个叫 StepMark 的小工具

> 写给所有经常"截图圈问题发出去"的人——产品、UI、实施、测试、医疗信息化同行。
> 这不是一篇技术文,讲的是"为什么会有这么个工具",以及它能帮你省下多少次鼠标点击。

---

## 先问一句:你平时怎么在截图上圈问题?

如果你和大多数人一样,流程大概是这样的:

1. 截屏(`Win+Shift+S` 或 QQ 截图);
2. 切到**矩形**工具,画个红框;
3. 切到**箭头**工具,补一根指向;
4. 切到**文字**工具,打一行说明;
5. 复制,贴到群里。

**一张图,切 3 次工具,点 7、8 下。**

要是图上有 3 个问题点,就是 9 次切换、20 多次点击。每天反馈 10 张图,一年点掉的鼠标公里数够绕地球一圈(夸张,但你会懂那种累)。

我做了很多年医疗信息化的需求评审和系统验收,每天都在重复这个动作。有一天终于受不了了,想:**为什么不能"拖一个框",就一步把红框 + 箭头 + 文字都生成出来?**

于是有了 **StepMark**。

---

## 它最不一样的地方:智能标注,一步到位

这是 StepMark 跟其他截图工具最大的区别,也是我最初想做它的原因。

**在截图上拖一个框,松手,它就帮你生成「红框 + 箭头 + 文字标签」三件套。**

![StepMark 智能标注效果](../images/hero.png)

具体怎么个"智能"法:

- 你**拖框的时候**,箭头会自动从**离你鼠标最近的那个角**射出来,跟着鼠标走;
- 你**点一下**确定箭头终点;
- 然后直接在弹出来的输入框里打字,标签就贴上去了。

也就是说,**一次拖拽 + 一次点击**,干完了传统流程的 3 次工具切换。

> 可能有人会问:这不是个小优化吗,值得专门做个工具?
> 值得。因为你每天会做这个动作几十次,任何一次"少切一次工具"都是真金白银的省心。

## 除了智能标注,还有这些顺手的能力

光有一个卖点撑不起一个工具,StepMark 在截图反馈这条线上做了完整的覆盖:

| 能力 | 解决什么 |
|------|----------|
| **🔢 自动序号标注** | 一张图标多个问题时,自动给每个标注编号(①②③…),读图的人一眼知道"先看哪个" |
| **🧰 完整工具集** | 智能标注 / 矩形 / 箭头 / 文字 / 马赛克,该有的都有 |
| **🎨 样式记忆** | 你喜欢细线就调细线,喜欢椭圆就调椭圆,每个工具记一套,下次还是这样 |
| **✂️ 可调裁剪区** | 进编辑器后发现截图范围不对,直接拖边缘调整,不用重新截 |
| **↩️ 撤销重做** | `Ctrl+Z` / `Ctrl+Y`,画错了就退回去 |
| **📤 多种导出** | 一键复制到剪贴板,或保存成 PNG / JPG |

**自动序号标注**这个尤其要单独说一下。

如果你做过验收或评审,会懂这个痛:一张图上圈了 5 个问题,群里发出去,开发问你"第 3 个是哪个?",你再截图标一遍……StepMark 会自动给每个标注编号,从智能标注到矩形/箭头/文字,全局递增,复制/保存都会带上。**问题清单这件事,图本身就能说话。**

## 为什么是"又一个截图工具"?

市面上的截图工具不少,我也不想再造一个轮子。做 StepMark 是因为——

- **Snipaste** 是我用过最好的截图工具之一,它的交互理念我非常认同。但它的标注偏"基础":矩形、箭头、画笔、文字是分开的,**没有"一步生成组合标注"**,也没有自动编号。
- **QQ / 微信截图**够轻,但标注能力更弱,且贴在聊天里,不适合归档。
- **专业修图软件**太重,不是为了"反馈问题"这个场景设计的。

StepMark 的定位很明确:**专门给"截图反馈问题"这个场景做的标注工具**。它对标 Snipaste 的经典交互(F1、框选、编辑器),但把"标注效率"做到了极致——**一步出三件套、自动编号、样式记忆**。

如果你日常的工作里,截图圈问题是高频动作,StepMark 值得一试。
